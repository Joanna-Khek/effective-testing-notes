# Test Driven Development
TDD serves as a guardrail to drive development, alleviates many hallucination issues.

It becomes more difficult when your code interacts with the outside world. 
User interfaces, network calls, databases, and time-based behavior all introduce complexity that is harder to control from a test.

## Loop
- [RED] Loop starts with red. Write test that expresses the behaviour you want
- [GREEN] Write enough code to make the test green.
- [Refactor] Once test passes, refactor. Improve the structure, remove duplication.
